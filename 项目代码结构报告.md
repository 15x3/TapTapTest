### 2.3 关键词+情感判断系统详解

**系统组成**:
1. **SentimentAnalyzer.lua** - 语气与关键词分析器
2. **ChatEventManager.lua** - 事件管理器（集成分析器）
3. **CSV options 字段** - 分支配置

**工作流程**:
```
用户输入文本 → SentimentAnalyzer.SelectBranch()
  ↓
1. 关键词精确匹配（优先级最高）
2. 语气分析匹配（次优先级）
3. 默认分支（default_next）
4. Fallback到第一个分支
  ↓
返回匹配的 nextId → ChatEventManager.JumpTo(nextId)
```

**CSV配置格式** (`chat_scenarios.csv:81`):
```csv
ch_3,示例5-关键词分支,0,choice,,,,,ch_a1,爬山,登山,运动,徒步,南山>ch_a1;海边,游泳,沙滩,玩水,冲浪>ch_b1;休息,不去,在家,懒得,不想动,算了>ch_c1,,ch_c1
```

**格式解析**:
```
"关键词1,关键词2,...,关键词N>跳转ID;关键词组2>跳转ID2;..."
```

**匹配示例**:
| 用户输入 | 匹配结果 | 跳转到 |
|---------|---------|--------|
| "我想去爬山" | 关键词"爬山" | `ch_a1` |
| "去海边游泳吧" | 关键词"海边"+"游泳" | `ch_b1` |
| "不想去了，懒得动" | 关键词"不想"+"懒得" | `ch_c1` |
| "好的" | 语气匹配 positive | 取决于配置 |

### 2.4 移除情感判断功能的编码建议

**目标**: 仅保留关键词匹配，移除语气分析

**修改文件**: `scripts/Utils/SentimentAnalyzer.lua`

**修改方案**:

```lua
-- === 方案1: 简化 SelectBranch 函数 ===

function SA.SelectBranch(userText, branches, defaultNextId)
    if not branches or #branches == 0 then
        return defaultNextId, defaultNextId and "default" or "none"
    end

    -- 仅保留关键词匹配
    local bestKeywordMatch = nil
    local bestKeywordScore = 0

    for _, branch in ipairs(branches) do
        local matched, score = SA.MatchKeywords(userText, branch.keywords)
        if matched and score > bestKeywordScore then
            bestKeywordScore = score
            bestKeywordMatch = branch.nextId
        end
    end

    if bestKeywordMatch then
        return bestKeywordMatch, "keyword"
    end

    -- 直接使用默认分支，不进行语气匹配
    if defaultNextId and defaultNextId ~= "" then
        return defaultNextId, "default"
    end

    -- Fallback
    if #branches > 0 then
        return branches[1].nextId, "default"
    end

    return nil, "none"
end

-- === 方案2: 完全移除情感分析模块（可选） ===

-- 可以删除以下内容以减小代码体积：
-- 1. POSITIVE_WORDS 词典
-- 2. NEGATIVE_WORDS 词典
-- 3. ANGRY_WORDS 词典
-- 4. QUESTION_WORDS 词典
-- 5. analyzePunctuation() 函数
-- 6. Analyze() 函数（如果不用于其他地方）

-- 但保留 MatchKeywords() 函数和 ParseBranchOptions() 函数
```

**简化后的优势**:
- 代码更简洁，逻辑更清晰
- 匹配规则更可预测
- 减少误匹配的可能性
- CSV配置更直观（不需要关心语气标签）
