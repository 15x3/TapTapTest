# 班主任晨间大作战 - 游戏设计文档

> 基于真实班主任工作素材的文字冒险游戏

---

## 游戏概述

**游戏名称：** 班主任晨间大作战
**游戏时长：** 10分钟（游戏内时间：8:00-8:10）
**游戏类型：** 文字冒险 / 模拟经营 / 信息筛选
**核心玩法：** 在不同群聊中收集消息，整理并转发到特定群，根据转发时间和关键词匹配度获得评分

---

## 游戏背景

玩家扮演一名高中班主任，在2026年3月18日周三早上8:00到8:00之间，需要处理来自各个群聊的海量消息。这些消息基于真实的班主任工作记录，包括：

- 学校通知（安全、心理普查、考试动员等）
- 家长咨询（请假、学习情况、食堂问题等）
- 学生问题（心理普查、证件丢失、日常询问等）
- 同事交流（设备报修、设施问题等）

玩家需要在有限的时间内，快速判断哪些信息需要转发，转发到哪个群，并合理分配注意力处理家长和学生的咨询。

---

## 核心机制

### 1. 信息筛选机制

玩家需要从海量信息中识别出需要转发的通知：

| 通知类型 | 转发目标 | 示例 |
|---------|---------|------|
| 学校重要通知 | 学生家长群 | 消防安全公开课、心理普查、防欺凌告家长书 |
| 纪律要求 | 学生班群 | 仪容仪表检查、时间管理、考场安排 |
| 活动通知 | 学生班群 | 社团纳新、比赛报名、会议通知 |
| 家长/学生咨询 | 原群回复 | 请假、学习问题、证件丢失等 |

**评分标准：**
- 正确识别需要转发的消息 → 获得「信息筛选」标签
- 错误转发或不转发重要消息 → 可能导致后续问题

---

### 2. 时间压力机制

游戏时间设定为8:00-8:10，共10分钟。真实时间与游戏时间比例为 **1:1**，即玩家有10分钟的实际游戏时间。

**时间节点：**
- 8:00 - 游戏开始，新手教程结束
- 8:01 - 学生家长群消息涌入
- 8:02 - 学校通知群消息涌入
- 8:03 - 学生班群消息涌入
- 8:04 - 突发事件（施工安全通知）
- 8:05 - 信息洪流（5条新消息同时到达）
- 8:06 - 交流群消息
- 8:07 - 多任务处理（6条消息同时到达）
- 8:08-8:10 - 最后冲刺与总结

**评分标准：**
- 在消息发出后2分钟内完成转发 → 获得「效率至上」标签
- 延迟转发可能影响某些情节发展

---

### 3. 情绪管理机制

玩家需要同时处理家长和学生的咨询，避免消息堆积。

**场景示例：**
- 家长询问孩子放学晚归问题
- 学生询问心理普查操作问题
- 家长申请请假
- 学生报告证件丢失

**评分标准：**
- 及时、耐心地回复所有咨询 → 获得「情绪管理」标签
- 忽略咨询或回复不及时 → 家长/学生可能多次追问

---

### 4. 标签系统

游戏通过TagManager记录玩家的表现，最终获得以下成就标签：

| 标签 | 获得条件 | 寓意 |
|-----|---------|------|
| 「新手」 | 完成新手教程 | 刚入门的班主任 |
| 「信息筛选」 | 正确识别并转发所有重要通知 | 具备良好的信息处理能力 |
| 「效率至上」 | 在2分钟内完成关键通知转发 | 高效的班主任 |
| 「情绪管理」 | 耐心回复所有家长和学生咨询 | 情绪稳定、沟通能力强 |
| 「时间压力」 | 在8:10前完成所有任务 | 时间管理能力强 |
| 「信息洪流」 | 成功应对8:05的信息洪流 | 抗压能力强 |
| 「危机处理」 | 快速转发施工安全通知 | 安全意识强 |
| 「上级压力」 | 完成所有学校通知转发 | 执行力强 |
| 「家校沟通」 | 妥善处理家长咨询 | 家校关系和谐 |
| 「学生管理」 | 妥善处理学生问题 | 关爱学生 |
| 「优秀班主任」 | 获得上述所有标签 | 完美班主任 |
| 「速通高手」 | 快速完成所有任务（可选隐藏成就） | 极致效率 |

## 功能特性

### 已使用的系统功能

| 功能 | 使用场景 | 示例 |
|-----|---------|------|
| `message` | 所有消息显示 | 系统提示、通知内容、聊天内容 |
| `typing` | 显示"正在输入" | 转发消息前的延迟效果 |
| `wait_input` + 自动输入 | 练习环节 | 新手教程中的自动输入练习 |
| `wait_input` + 自由输入 | 回复家长/学生 | 请假批准、问题解答 |
| `choice` | 选择转发目标/回复方式 | 是否转发、优先级选择 |
| `set_time` | 设置游戏开始时间 | 从8:00开始 |
| `freeze_time` | 新手教程冻结时间 | 完成教程后解冻 |
| `tag` | 记录玩家表现 | 获得成就标签 |
| `require_tag` | 条件触发（预留） | 后续可扩展分支剧情 |
| `thresholds` | 阈值分支（预留） | 后续可扩展多级评分 |

## 游戏流程

### 阶段1：新手教程（时间冻结）

**目标：** 教会玩家基本操作和规则

**教学内容：**
1. 核心玩法说明
2. 评分规则介绍
3. 操作指南（自动输入/手动输入）
4. 练习任务（选择正确的转发目标）
5. 自动输入练习

**完成条件：** 通过选择题测试

**奖励：** 获得「新手」标签

---

### 阶段2：8:00-8:04 逐渐升温

**8:00 - 班主任通知群**
- 收到防欺凌告家长书通知
- 需要转发到学生家长群
- **考点：** 识别需要转发的通知

**8:01 - 学生家长群**
- 历史消息（交通安全、身份证上传）
- 家长咨询（食堂饭菜、放学晚归）
- **考点：** 区分历史消息和新消息，选择是否回复

**8:02 - 学校通知群**
- 仪容仪表检查通知（需要转发到学生班群）
- 新书领取通知（无需处理，不相关）
- 心理普查通知（需要转发到学生班群）
- **考点：** 筛选相关通知，识别转发目标

**8:03 - 学生班群**
- 学生讨论（新课本、食堂饭菜）
- 学生咨询（身份证丢失、心理普查操作）
- **考点：** 区分闲聊和需要回复的问题，选择回复方式（单独/统一）

---

### 阶段3：8:04 突发事件

**事件：**
- 校史馆广场施工安全通知
- 家长请假申请

**考点：**
- 快速识别紧急安全通知
- 立即转发到学生班群
- 及时处理家长请假

**奖励：** 快速转发获得「效率至上」+「危机处理」标签

---

### 阶段4：8:05 信息洪流

**事件：** 5条新消息同时到达班主任通知群

1. 班级资产明细表领取（行政任务）
2. 住宿生告家长书（需要转发）
3. 心理筛查进度提醒（跟进）
4. 学考动员大会通知（需要转发）
5. ...（第5条消息）

**考点：**
- 快速识别需要转发的通知
- 过滤行政任务和跟进提醒
- 在有限时间内完成转发

**奖励：** 快速筛选获得「信息洪流」+「效率至上」标签

---

### 阶段5：8:06-8:07 多任务处理

**8:06 - 交流群**
- 同事设备报修交流
- **考点：** 识别无关消息，无需处理

**8:07 - 多任务挑战**
- 学生家长群：3条新消息
- 学生班群：2条新消息
- 班主任通知群：1条助学金通知
- **考点：** 优先级排序，优先处理重要通知

**奖励：** 正确优先级获得「信息筛选」标签

---

### 阶段6：8:08-8:10 总结评价

**最后任务：**
- 民族管弦乐团纳新通知（可选转发）

**游戏结束：**
- 显示游戏结束提示
- 统计获得的标签
- 感谢游玩

## 真实素材还原

### 基于真实聊天记录

游戏中的所有通知内容、对话风格、消息格式均来自真实的班主任工作群聊天记录，包括：

1. **通知内容**
   - 防欺凌告家长书（王丹妮，安全保卫处）
   - 仪容仪表检查（苏仁辉，政教处）
   - 心理普查未测名单（许奕蓉，心理辅导室）
   - 学考动员大会（邓星妹，教务处）

2. **聊天风格**
   - 家长：礼貌询问、@所有人、表情符号
   - 学生：随意、网络用语、@老师
   - 学校领导：正式通知、格式规范、强调截止时间
   - 同事：简短、直接、工作相关

3. **时间压力**
   - 多个任务同时截止（3月20日、3月27日、3月29日）
   - 截止时间强调（下午17:00前、本周五下班前）
   - 紧急通知（务必重视、确保不漏一人）

## 文件结构

### 数据文件

```
scripts/data/
├── chat_scenarios_game.csv    # 游戏对话事件（200+ 行）
├── chats_game.csv             # 聊天列表配置（6个群）
├── chat_scenarios_backup.csv  # 原始文件备份
└── [原始CSV文件保留]
```

### CSV字段说明

**chat_scenarios_game.csv：**
- `id`: 事件唯一标识
- `chat_match`: 聊天群匹配（`*` = 任意群）
- `delay`: 延迟秒数
- `type`: 事件类型（message/typing/wait_input/choice/set_time/freeze_time）
- `sender`: 发送者
- `text`: 消息内容
- `next`: 下一个事件ID
- `options`: 分支选项
- `timeout`: 超时秒数
- `default_next`: 默认跳转
- `tag`: 获得的标签
- `thresholds`: 阈值分支
- `require_tag`: 前置标签条件

**chats_game.csv：**
- `name`: 群名称
- `msg`: 预览消息
- `tag`: 群标签
- `tag_color`: 标签颜色
- `badge`: 未读消息数
- `icon_color`: 图标颜色
- `icon_text`: 图标文字
- `time`: 最后消息时间

## 使用方法

### 1. 替换数据文件

```bash
# 备份原始文件
cp scripts/data/chat_scenarios.csv scripts/data/chat_scenarios_backup.csv
cp scripts/data/chats.csv scripts/data/chats_backup.csv

# 使用游戏文件
cp scripts/data/chat_scenarios_game.csv scripts/data/chat_scenarios.csv
cp scripts/data/chats_game.csv scripts/data/chats.csv
```

### 2. 启动游戏

1. 运行游戏主程序
2. 在主屏幕找到【新手教程】聊天
3. 点击进入新手教程
4. 完成教程后，时间开始流动（8:00）
5. 在不同群之间切换，处理消息

### 3. 查看标签

游戏结束后，可以通过以下方式查看获得的标签：

```lua
-- 在游戏中查看
print(TagManager.GetAll())

-- 或在控制台查看
-- TagManager会自动打印获得的标签
```

## 教育意义

### 游戏反映的真实问题

1. **信息过载**
   - 班主任每天需要处理数十甚至上百条消息
   - 多个群聊同时涌入，难以兼顾

2. **时间压力**
   - 大量任务集中在早上和放学时段
   - 截止时间紧迫，难以从容处理

3. **多任务处理**
   - 同时应对学校、家长、学生三方需求
   - 需要快速判断优先级

4. **情绪管理**
   - 家长的不理解和抱怨
   - 学生的各种突发问题
   - 学校的行政压力

5. **责任重大**
   - 安全问题不能拖延
   - 心理问题需要关注
   - 学习任务需要督促

### 对玩家 empathy 的培养

通过游戏，玩家可以：
- 体验班主任工作的真实压力
- 理解为什么班主任有时回复不及时
- 认识到教育工作的复杂性和重要性
- 学会在信息洪流中保持冷静和理性

## 测试建议

### 测试要点

1. **新手教程**
   - 是否清晰易懂
   - 是否覆盖所有操作
   - 是否有遗漏的功能

2. **时间节奏**
   - 10分钟是否合适
   - 是否需要调整时间流速
   - 各时段消息量是否平衡

3. **难度曲线**
   - 8:00-8:04 是否过于简单
   - 8:05 信息洪流是否过于困难
   - 整体难度是否循序渐进

4. **标签系统**
   - 标签获得条件是否合理
   - 标签名称是否准确
   - 是否有遗漏的标签

5. **真实性**
   - 消息内容是否真实可信
   - 对话风格是否符合实际
   - 是否反映班主任工作特点

### 测试方法

1. **完整测试**
   - 从头到尾玩一遍
   - 记录所有问题和建议
   - 统计获得的标签

2. **压力测试**
   - 快速点击，测试系统稳定性
   - 忽略所有消息，查看后果
   - 故意延迟转发，体验时间压力

3. **对比测试**
   - 多人同时测试，对比体验
   - 收集不同玩家的反馈
   - 分析难点和痛点

## 常见问题

**Q: 为什么有些消息不需要转发？**
A: 真实工作中，不是所有消息都需要转发。玩家需要判断消息的重要性、紧急性和相关性。这是信息筛选能力的体现。

**Q: 如果我忽略了所有消息会怎样？**
A: 游戏不会强制失败，但会失去获得标签的机会，可能影响最终评价。

**Q: 标签有什么用？**
A: 标签是游戏的主要成就系统，也是玩家表现的量化体现。后续可以扩展为解锁隐藏内容、影响结局等。

**Q: 游戏时间可以调整吗？**
A: 可以通过修改CSV中的delay字段调整时间节奏，或通过GameTime模块修改时间流速。

**Q: 可以添加更多聊天群吗？**
A: 可以，在chats.csv中添加新的群，在chat_scenarios.csv中添加对应事件即可。
