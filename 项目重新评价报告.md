## 一、问题诊断与根因分析

### 问题1：核心玩法未能完整复现

#### 用户反馈
> 虽然已实现随意敲打键盘的功能，但是敲打后会自动发送，而不是应该在完整出现选项后回车才能发送

#### 实际情况分析

**❌ 当前项目并未实现"随意敲打键盘"功能**

经过深入代码审查，发现以下事实：

1. **无键盘事件监听**
   - `ChatEventManager.lua` 中没有任何 `KeyDown`/`KeyPress` 事件处理
   - `DingtalkPages.lua` 和 `WechatPages.lua` 中没有全局键盘监听

2. **无自动填充逻辑**
   - CSV数据结构中没有 `expected_reply` 或类似字段
   - 数据层（`DingtalkData.lua`/`WechatData.lua`）中没有预设回复的定义

3. **输入系统是标准的TextField**
   ```lua
   -- DingtalkPages.lua 第942-950行
   inputField = UI.TextField {
       flexGrow = 1,
       height = 34,
       fontSize = 12,
       placeholder = "输入消息...",
       onSubmit = function(self, value)
           sendMessage(value)  -- 仅回车时发送
       end,
   }
   ```

#### 用户误解的可能原因

用户可能将**回车快捷发送**误解为"随意敲打自动发送"：

- ✅ 当前实现：按回车键（或点击发送按钮）→ 发送消息
- ❌ 《S4U》玩法：随意敲击任意键 → 自动填充预设回复 → 按回车发送

#### 《S4U》核心玩法的正确实现方式

```
正确的交互流程：
┌─────────────────────────────────────────────────────────┐
│  1. 系统进入 wait_input 状态                            │
│  2. 显示提示："按任意键开始输入"                        │
│  3. 玩家随意敲击键盘（A/S/D/F/任意键）                   │
│  4. 系统自动填充预设回复（逐字显示动画）                 │
│  5. 输入框显示完整预设回复                               │
│  6. 玩家按回车键（或点击发送）→ 发送消息                │
│  7. 系统恢复对话流程                                     │
└─────────────────────────────────────────────────────────┘
```

#### 实现方案建议

**方案1：数据层增强**

在CSV中增加 `expected_reply` 字段：

```csv
id,chat_match,delay,type,sender,text,time,show_time,next,options,timeout,default_next,expected_reply
a_4,示例2-等待回复,0,wait_input,,,,no,,,,收到
```

**方案2：事件管理器增强**

在 `ChatEventManager.lua` 中添加：

```lua
--- 新增字段
self.expectedReply = nil  -- 当前期待的预设回复

--- 在 wait_input 事件时加载预设回复
elseif eventType == "wait_input" then
    self.expectedReply = event.expected_reply or ""
    self.state = "wait_input"

    -- 通知UI：进入随意输入模式
    if self.callbacks.onWaitInput then
        self.callbacks.onWaitInput(self.expectedReply)
    end
    return false
end

--- 新增：处理任意按键
function Manager:OnAnyKey()
    if self.state ~= "wait_input" then return end
    if not self.expectedReply or self.expectedReply == "" then return end

    -- 自动填充输入框
    if self.callbacks.onAutoFill then
        self.callbacks.onAutoFill(self.expectedReply)
    end
end
```

**方案3：页面层键盘监听**

在 `DingtalkPages.lua` / `WechatPages.lua` 中添加：

```lua
-- 在页面创建时订阅键盘事件
SubscribeToEvent("KeyDown", "HandleChatKeyDown")

function HandleChatKeyDown(eventType, eventData)
    local key = eventData["Key"]:GetInt()

    -- 忽略特殊键（回车、退格等）
    if key == KEY_RETURN or key == KEY_BACKSPACE then
        return
    end

    -- 在wait_input状态下，任意键触发自动填充
    if activeManager_ and activeManager_.state == "wait_input" then
        -- 防止重复触发
        if not autoFillTriggered_ then
            autoFillTriggered_ = true
            local expectedReply = activeManager_.expectedReply

            -- 逐字动画填充
            if inputField and expectedReply and expectedReply ~= "" then
                AnimateAutoFill(inputField, expectedReply)
            end
        end
    end
end

--- 逐字填充动画
function AnimateAutoFill(textField, targetText, callback)
    local currentLength = 0
    local totalLength = #targetText
    local charInterval = 0.05  -- 每个字符间隔50ms

    -- 创建定时器
    local timer = 0
    local animHandle = SubscribeToEvent("Update", function(_, ev)
        timer = timer + ev["TimeStep"]:GetFloat()

        if timer >= charInterval then
            timer = 0
            currentLength = currentLength + 1

            if currentLength <= totalLength then
                -- 填充到当前长度
                textField:SetValue(string.sub(targetText, 1, currentLength))
            else
                -- 动画完成
                UnsubscribeFromEvent("Update", animHandle)
                if callback then callback() end
            end
        end
    end)
end

-- 发送时重置标志
local function sendMessage(text)
    -- ... 原有逻辑 ...

    -- 重置自动填充标志
    autoFillTriggered_ = false

    -- 通知事件管理器
    if activeManager_ and activeManager_:IsWaitingInput() then
        activeManager_:OnUserMessage(trimmed)
    end
end
```

### 问题2：用户对话框文字未渲染 🔴 关键Bug

#### 问题定位

**根因：TextField 组件缺少文字颜色设置**

检查所有 TextField 定义：

**钉钉聊天页面**（`DingtalkPages.lua` 第942-950行）：
```lua
inputField = UI.TextField {
    flexGrow = 1,
    height = 34,
    fontSize = 12,
    placeholder = "输入消息...",
    -- ❌ 缺少 textColor/fontColor 设置！
    -- ❌ 导致文字颜色可能为透明或与背景色相同
    onSubmit = function(self, value)
        sendMessage(value)
    end,
}
```

**微信聊天页面**（`WechatPages.lua` 第251-259行）：
```lua
local textField = UI.TextField {
    flexGrow = 1, flexBasis = 0,
    height = 36,
    fontSize = 13,
    placeholder = "输入消息...",
    value = "",
    borderRadius = 6,
    backgroundColor = WX.white,  -- ✅ 设置了背景色
    paddingHorizontal = 10,
    -- ❌ 但仍然缺少 textColor/fontColor！
    onChange = function(self, value)
        inputValue = value
        -- ...
    end,
}
```

#### 对比其他UI组件

**Label 组件都有颜色设置：**
```lua
UI.Label {
    text = msg.text,
    fontSize = 11,
    fontColor = WX.text,  -- ✅ 有颜色
    whiteSpace = "normal",
}
```

**TextField 组件缺少颜色：**
```lua
UI.TextField {
    fontSize = 12,
    -- ❌ 缺少 fontColor/textColor
}
```

#### 修复方案

**方案A：显式设置颜色**

function DingtalkData.GetChatScenario(chatName)
    if scenarioCache_[chatName] then
        return scenarioCache_[chatName]
    end
    -- 按需加载
    local data = LoadCSV("data/scenarios/" .. chatName .. ".csv")
    scenarioCache_[chatName] = data
    return data
end

-- 2. 对象池复用气泡
local bubblePool_ = {}
function GetBubbleFromPool()
    if #bubblePool_ > 0 then
        return table.remove(bubblePool_)
    end
    return CreateNewBubble()
end

function ReturnBubbleToPool(bubble)
    bubble:SetVisible(false)
    table.insert(bubblePool_, bubble)
end
