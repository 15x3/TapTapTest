# 一日班主任 —— 文案设计文档

> **版本**: v1.0
> **日期**: 2026-03-27
> **状态**: 初版，待文案评审

---

## 1. 游戏基调

### 1.1 一句话描述

模拟班主任在课间 10 分钟内处理手机消息的生存游戏。

### 1.2 情感基调

**吐槽向、轻松共鸣，不煽情。**

这个游戏的核心情绪是"忙到想笑"——就像你跟朋友吐槽工作日常，讲的时候自己都觉得好笑。玩家不需要被感动，不需要被教育，只需要在手忙脚乱中会心一笑："可不就是这样吗。"

### 1.3 不做的事情

| 不做 | 原因 |
|------|------|
| 不做深度情感叙事 | 玩家不需要情感负担 |
| 不做道德说教 | 不是为了说"老师很辛苦" |
| 不做积分/评分系统 | 教育行业的绩效不是积分能衡量的 |
| 不美化/丑化任何群体 | 家长、学生、领导都是正常人 |

### 1.4 语言风格

- 像真实聊天记录，不像文学作品
- 用语贴近日常，允许口语化、网络用语
- 家长群的氛围要真实：有客气的、有啰嗦的、有沉默的
- 学生群的氛围要真实：有活跃的、有只发表情包的
- 领导/同事的语气要真实：有公事公办的、有随和的

## 2. 世界设定

### 2.1 学校

- **学校名称**：（待定，建议取一个普通但有辨识度的名字）
- **学段**：初中（七年级）
- **班级**：七年三班
- **玩家身份**：七年三班班主任，兼任某学科教师

### 2.2 平台设定

游戏发生在一个手机屏幕内，玩家使用两个通讯软件：

| App 名称 | 原型 | 用途 |
|---------|------|------|
| **叮叮** | 钉钉 | 学校工作沟通（年级群、工作群、领导私聊） |
| **微言** | 微信 | 家校沟通 + 学生沟通（家长群、班级群、私聊） |

这两个平台的分野很重要：**工作通知从叮叮来，需要转发到微言的家长群/班级群**。这是信息流转的核心路径。

## 3. 角色设定

### 3.1 设计原则

每个角色不需要完整的人物小传，但需要有**明确的说话风格**和**在信息流中的功能定位**。

### 3.2 叮叮平台角色（工作侧）

| 角色 | 身份 | 说话风格 | 功能定位 |
|------|------|---------|---------|
| **刘主任** | 年级主任 | 简洁公事，偶尔用感叹号强调。"各位班主任，请务必今天完成！" | 信息源头，重要通知从他这里发出 |
| **张主任** | 教务处副主任 | 客气但有压力感。"老师辛苦了，体检表麻烦下午前交一下哈" | 私聊布置任务，制造额外事项 |
| **周老师** | 隔壁班班主任 | 随和，偶尔在群里接话。"我们班也是，哈哈" | 工作群闲聊噪音 / 偶尔有用信息 |
| **陈老师** | 数学组老师 | 喜欢在群里分享教学资源。"分享一篇好文章" | 噪音来源 |

### 3.3 微言平台角色（家校侧）

| 角色 | 身份 | 说话风格 | 功能定位 |
|------|------|---------|---------|
| **李妈妈** | 家委会成员 | 热情，爱发早安图和鸡汤。"老师早上好[太阳]" | 典型噪音来源，但偶尔转发有用信息 |
| **王爸爸** | 普通家长 | 沉默型，偶尔冒泡就是有正事。"老师，孩子说眼镜坏了" | 低频但重要的信息来源 |
| **赵妈妈** | 爱操心的家长 | 喜欢追问细节。"老师，调课是哪天？具体几点？" | 制造追问压力 |
| **张家长** | 较少出现 | 跟风型。"收到" "好的" "+1" | 正面反馈的主要来源 |
| **小明** | 班长 | 认真负责，用语较正式。"老师，XX 和 XX 打架了" | 学生侧的重要信息来源 |
| **小红** | 普通学生 | 活泼，爱发表情包 | 学生群噪音 |

### 3.4 角色数量控制

第一关建议控制在 **8-10 个活跃角色**，避免玩家信息过载。部分角色只出现名字（如家长群里其他回复"收到"的家长）不需要完整设定。

## 4. 第一关：课间十分钟

### 4.1 情境简报

> "第二节课刚下课。你有 10 分钟处理消息，第三节课就要上课了。"

简报出现在关卡开始前，全屏展示，玩家点击"开始"后进入。简报要求：简短、交代清楚时间压力，不要铺垫过多背景。

### 4.2 关卡信息地图

本关需要玩家处理的核心信息链条：

```
信息链 A：调课通知
  来源：刘主任 → 年级工作群（叮叮）
  玩家操作：转发 → 七年三班家长群（微言）
  反馈：家长回复"收到"

信息链 B：体检表催收
  来源：张主任 → 私聊（叮叮）
  玩家操作：回复张主任（确认收到/说明进度）
  反馈：张主任回复"好的辛苦了"

信息链 C：体检注意事项
  来源：刘主任 → 年级工作群（叮叮）
  玩家操作：转发 → 七年三班家长群（微言）
  反馈：家长回复"收到"

信息链 D：家长私聊求助
  来源：王爸爸 → 私聊（微言）
  玩家操作：回复王爸爸
  反馈：王爸爸回复"谢谢老师，麻烦了"

信息链 E：学生突发事件
  来源：小明 → 私聊（微言）
  玩家操作：回复小明
  反馈：小明回复"好的老师，我知道了"

终局：发布公告
  目标群：七年三班班级群（微言）
  关键词：调课、体检
  玩家操作：自由输入公告内容
```

### 4.3 消息时间轴

以下是第一关完整的消息编排。时间为关卡开始后的秒数（0-600，即 0:00-10:00）。

**说明**：
- **[重要]** 标记的消息需要玩家操作
- **[噪音]** 标记的消息应该被忽略
- **[反馈]** 标记的消息由系统根据玩家行为动态触发

#### 关卡结束：发布公告

时间到后（或玩家主动发起），弹出公告编辑界面，玩家在**七年三班班级群**发布公告。

**关键词检测要求**：

| 信息组 | 必须提及的关键词（任一即可） | 加分细节关键词 |
|--------|--------------------------|---------------|
| 调课通知 | 调课 / 换课 / 自习 | 周三、下午、第六节、第七节 |
| 体检通知 | 体检 | 周五、宽松衣物、不带首饰 |

**公告质量示例**：

优秀公告（玩家输入）：
> "同学们注意，明天（周三）下午第六七节调课为自习。另外周五下午全校体检，请穿宽松衣物，不要佩戴首饰。"

及格公告：
> "明天下午调课了，另外周五体检。"

遗漏公告：
> "明天调课了，大家知道一下。"（遗漏了体检信息）

## 5. 反馈消息文案模板

### 5.1 正面反馈（玩家正确操作后触发）

**转发通知后，家长群反应**：

```
李妈妈：收到，谢谢老师~
张家长：收到
赵妈妈：好的老师，周三下午是吧
王爸爸：收到了，谢谢
```

**回复家长私聊后**：

```
王爸爸：好的老师，麻烦您了
王爸爸：谢谢老师关心
```

**回复张主任后**：

```
张主任：好的辛苦了~
张主任：[OK手势]
```

**回复学生后**：

```
小明：好的老师，我知道了
小明：我会注意的
```

### 5.2 负面反馈（玩家超时未处理 / 操作错误时触发）

**长时间未转发调课通知（超过 120 秒）**：

```
-- 年级群
刘主任：@你 调课通知转发家长了吗？

-- 或私聊
刘主任：老师，调课的事情麻烦尽快通知家长哈
```

**长时间未回复张主任（超过 90 秒）**：

```
张主任：老师？在吗
张主任：体检表的事情方便回复一下吗
```

**长时间未回复学生（超过 90 秒）**：

```
小明：老师？
小明：老师你在吗，张同学他们又开始了
```

**转发到错误的群**：

```
-- 如果把工作群内部对话转到了家长群
赵妈妈：老师，这个是发给我们看的吗？
李妈妈：？

-- 如果把家长群消息转到了年级工作群
周老师：这是谁发错群了？哈哈
```

### 5.3 公告反馈

公告发布后，根据关键词检测结果，班级群内会有不同反应：

**信息完整**：
```
小明：收到老师
小红：[OK]
学生A：收到
```

**遗漏了信息**：
```
小明：收到老师
-- （没有额外反应，遗漏的信息在结算报告中体现）
```

## 6. 信息处理报告文案

### 6.1 格式规范

报告以"事实陈述"为主，不做情感评价，不给分数。

### 6.2 模板

```
┌─────────────────────────────────┐
│        信息处理报告              │
│                                  │
│  ✅ 已处理                       │
│  · 调课通知 → 已转发至家长群     │
│  · 王爸爸来信 → 已回复           │
│  · 张主任询问 → 已回复           │
│                                  │
│  ❌ 遗漏                         │
│  · 体检注意事项 → 未转发         │
│  · 小明反映情况 → 未回复         │
│                                  │
│  📋 公告                         │
│  · ✅ 提到：调课                 │
│  · ❌ 遗漏：体检                 │
│                                  │
│       [再来一次]   [结束]        │
└─────────────────────────────────┘
```

### 6.3 文案原则

- 用"已处理/遗漏"而非"正确/错误"
- 用具体事项描述而非抽象评语
- 不说"做得好"或"需要改进"，让玩家自己判断
- 公告部分只列出"提到了什么/遗漏了什么"

## 8. 写作注意事项

### 8.1 给文案同学的备注

1. **真实感是最重要的**。每条消息都想想"这像不像真的会出现在群里的消息"。
2. **不要过度设计冲突**。这不是剧情游戏，戏剧性来自信息量本身，不需要刻意制造矛盾。
3. **噪音消息很重要**。它们不是凑数的，是让"筛选"这个动作有意义的关键元素。没有噪音，就没有筛选。
4. **每个角色的发言风格要一致**。李妈妈永远热情带表情，王爸爸永远简短直接，不要串。
5. **消息长度要自然**。工作通知可以长一点（2-3 行），家长闲聊通常很短（1 行），学生更短。
6. **时间密度要有节奏**。不要均匀分布消息，要有疏有密，让玩家有喘息也有高峰。

### 8.2 后续扩展方向（备忘）

如果做更多关卡，可以考虑以下场景：

| 场景 | 特殊元素 |
|------|---------|
| 开学第一天 | 大量新家长入群、自我介绍刷屏 |
| 考试周 | 家长追问成绩、学生情绪波动 |
| 家长会前 | 准备材料、通知时间地点 |
| 运动会 | 报名、分工、安全通知 |
| 校园突发事件 | 学生受伤、设施损坏 |
| 学期末 | 评优、成绩单、假期通知爆炸 |
